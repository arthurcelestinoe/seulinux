Glossário Linux

Termos técnicos, sem complicação.

Consulte explicações rápidas para palavras que aparecem em guias, fóruns e resultados do teste. As definições são introdutórias: o objetivo é ajudar você a continuar a leitura sem perder o contexto.

01

Comece por aqui

Fundamentos

Linux
Em sentido estrito, é o kernel. No uso cotidiano, “Linux” também descreve sistemas completos construídos ao redor dele, como Ubuntu, Fedora e Linux Mint.
Kernel
É o núcleo do sistema. Faz a ponte entre programas e hardware, gerenciando memória, processador, dispositivos e processos.
Distribuição (distro)
Um sistema completo que reúne o kernel Linux, programas, instalador, atualizações e decisões próprias de configuração.
Ambiente gráfico
O conjunto que forma a experiência visual: painel, menu, janelas, configurações e aplicativos básicos. GNOME, KDE Plasma e Xfce são exemplos.
Software livre / código aberto
Software cujo código pode ser estudado e, conforme a licença, modificado e redistribuído. “Livre” se refere a liberdades, não necessariamente a preço.
LTS
“Suporte de longo prazo”. Versão mantida por mais tempo, geralmente voltada a quem prefere estabilidade e menos mudanças.
Rolling release
Modelo que recebe atualizações contínuas, sem depender de grandes reinstalações entre versões. Traz software recente, mas pede acompanhamento maior.
Sessão live
Permite iniciar uma distribuição pelo pendrive para testar interface e hardware sem instalá-la no disco.
02

Por baixo da interface

Sistema e permissões

Terminal
Aplicativo que oferece uma interface de texto para executar comandos. É uma janela; o programa que interpreta os comandos é o shell.
Shell
Programa que interpreta comandos e executa tarefas. Bash e Zsh são shells comuns. Também pode ser usado em scripts de automação.
Diretório
O equivalente a uma pasta. Diretórios organizam arquivos em uma hierarquia que começa em /, chamado diretório raiz.
Root
Pode significar o usuário administrador, com acesso total, ou o diretório raiz /. O contexto indica qual dos dois está sendo mencionado.
sudo
Executa um comando com privilégios administrativos, após autenticação. Deve ser usado com atenção porque permite alterar partes críticas do sistema.
Permissões
Regras que determinam quem pode ler, modificar ou executar um arquivo. Normalmente são separadas entre dono, grupo e outros usuários.
Processo
Uma instância de um programa em execução. Cada processo recebe um identificador, chamado PID, e usa recursos como memória e processador.
Daemon
Programa que funciona em segundo plano oferecendo um serviço, como rede, impressão ou agendamento. Geralmente inicia junto com o sistema.
03

Instalação e atualização

Programas e pacotes

Pacote
Arquivo preparado para instalar um programa, biblioteca ou componente, incluindo informações de versão e dependências.
Gerenciador de pacotes
Ferramenta que instala, atualiza e remove pacotes com segurança. APT, DNF, Pacman e Zypper são exemplos.
Repositório
Fonte organizada de pacotes mantida pela distribuição ou por terceiros. O gerenciador consulta repositórios para encontrar programas e atualizações.
Dependência
Outro pacote necessário para um programa funcionar. O gerenciador de pacotes normalmente resolve e instala dependências automaticamente.
Flatpak
Formato de aplicativo que funciona em várias distribuições e isola parte do programa do sistema. Flathub é seu repositório mais conhecido.
Snap
Formato de pacote criado pela Canonical, usado principalmente no Ubuntu. Inclui dependências e recebe atualizações automáticas.
AppImage
Aplicativo distribuído como um único arquivo executável. Em muitos casos não exige instalação, mas atualizações podem ser manuais.
Codec
Componente que codifica ou decodifica áudio e vídeo. Algumas distribuições não incluem certos codecs por padrão devido a licenças ou patentes.
04

O que aparece na tela

Interface gráfica

GUI
“Interface gráfica do usuário”. Permite interagir com janelas, botões, menus e ícones, em contraste com uma interface baseada somente em texto.
Wayland
Protocolo gráfico moderno usado para exibir janelas e receber entradas de teclado e mouse. Busca mais segurança e melhor suporte a telas atuais.
X11 / X.Org
Sistema gráfico tradicional do Linux, maduro e amplamente compatível. Está sendo gradualmente substituído pelo Wayland.
Compositor
Organiza e desenha as janelas na tela, cuidando de efeitos, transparência e sincronização. KWin e Mutter são exemplos.
Gerenciador de janelas
Controla posição, tamanho e comportamento das janelas. Pode fazer parte de um ambiente gráfico ou ser usado sozinho.
HiDPI
Tela com alta densidade de pixels, comum em notebooks e monitores 4K. Pode exigir escala para textos e controles não ficarem pequenos.
05

Do botão de ligar ao desktop

Hardware e inicialização

Driver
Software que permite ao sistema conversar com um componente de hardware, como GPU, Wi-Fi, impressora ou touchpad.
Firmware
Código de baixo nível usado por um dispositivo. Algumas placas precisam que o sistema carregue um arquivo de firmware para funcionar corretamente.
BIOS / UEFI
Firmware da placa-mãe que prepara o hardware e inicia o sistema operacional. UEFI é o padrão moderno.
Boot
Processo de inicialização, desde ligar o computador até o sistema ficar disponível para uso.
Bootloader
Programa que encontra e inicia o sistema operacional. Também pode oferecer um menu para escolher entre Linux e Windows. GRUB é um exemplo.
Dual boot
Configuração com dois sistemas instalados no mesmo computador. A escolha é feita ao ligar; exige cuidado com partições e backup.
Máquina virtual
Computador simulado dentro de outro sistema. É útil para testar uma distribuição sem alterar o disco, embora tenha desempenho inferior ao uso direto.
06

Armazenamento

Discos e arquivos

Sistema de arquivos
Formato usado para organizar dados em uma partição. ext4 e Btrfs são comuns no Linux; NTFS é comum no Windows.
Partição
Divisão lógica de um disco. Cada partição pode guardar um sistema de arquivos ou cumprir uma função específica.
Montar
Tornar um sistema de arquivos acessível em determinado diretório. Pendrives normalmente são montados automaticamente.
/home
Local onde ficam os diretórios pessoais dos usuários, incluindo documentos e muitas configurações de aplicativos.
Swap
Espaço em disco usado como apoio à memória RAM e, em algumas configurações, para hibernação. É mais lento que RAM.
Snapshot
Registro do estado de um sistema de arquivos em certo momento. Pode ajudar a desfazer alterações, mas não substitui um backup externo.
